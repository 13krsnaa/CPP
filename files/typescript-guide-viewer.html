Glossary

TypeScript ke terms simple bhasha mein. Confuse ho jao tab yahan aao.

Interface vs Type — Sabse Common Confusion

Interface
interface User { name: string age: number } // Objects ke liye best // Extend ho sakta hai // Declaration merging hoti hai
Type Alias
type User = { name: string age: number } // Har cheez ke liye kaam karta hai // Union/Intersection kar sakte ho // type ID = string | number

Simple rule: Objects ke liye interface, baaki ke liye type

Important Terms

any"Mujhe type nahi pata" — TypeScript ki sabse buri cheez. Avoid karo.
unknownany se safe version — use karne se pehle type check zaroori
voidFunction kuch return nahi karta — function log(): void
neverYe code kabhi nahi chalega — exhaustive checks mein
optional (?)Field zaroori nahi — name?: string
readonlyChange nahi kar sakte — readonly id: number
Generic (T)"Koi bhi type chalega, but consistent" — Array<string>
Union (|)"Ye ya wo" — string | number
Intersection (&)"Ye bhi aur wo bhi" — User & Admin
@types/JS libraries ke type definitions — @types/express
tsconfig.jsonTypeScript ka config file — project root mein
.ts / .tsx.ts = backend, .tsx = React components

TS Basics Deep Dive

Interface, Type, Generic, Utility Types — sab kuch MERN developer ki nazar se.

Utility Types — Sabse Kaam Ka Section

typescript
interface User { id: number name: string email: string password: string } // Partial — sab optional (PATCH API ke liye perfect) type UpdateUser = Partial<User> // Pick — sirf ye fields lo (public profile) type PublicUser = Pick<User, 'id' | 'name' | 'email'> // Omit — ye field hata do (POST ke liye, id auto-generate hoga) type CreateUser = Omit<User, 'id'> // POST /users — id nahi chahiye function createUser(userData: CreateUser): User { return { id: Math.random(), ...userData } } // PATCH /users/:id — sirf kuch fields function updateUser(id: number, updates: Partial<Omit<User, 'id'>>): User { // update logic }

Generic — Ek Baar Likhao, Har Jagah Use Karo

typescript
// Without generic — repeat karna padta hai function getFirstNumber(arr: number[]): number { return arr[0] } function getFirstString(arr: string[]): string { return arr[0] } // With generic — ek function, sab types ke liye function getFirst<T>(arr: T[]): T { return arr[0] } getFirst<number>([1, 2, 3]) // returns number getFirst<string>(["a", "b"]) // returns string // API Response — real world use interface ApiResponse<T> { success: boolean data: T message: string } type UserResponse = ApiResponse<User> type ProductsResponse = ApiResponse<Product[]>

MongoDB + Mongoose TS

MERN developers ke liye sabse important part — Mongoose models properly typed.

Model Banana — Sahi Tarika

typescript — models/user.model.ts
import { Schema, model, Document } from 'mongoose' // Step 1: Data interface export interface IUser { name: string email: string password: string role: 'admin' | 'user' } // Step 2: Document type (Mongoose methods ke saath) export interface IUserDocument extends IUser, Document { comparePassword(password: string): Promise<boolean> } // Step 3: Schema with type const userSchema = new Schema<IUserDocument>({ name: { type: String, required: true }, email: { type: String, required: true, unique: true }, password: { type: String, required: true, select: false }, role: { type: String, enum: ['admin', 'user'], default: 'user' } }, { timestamps: true }) export const User = model<IUserDocument>('User', userSchema)

Common Patterns

typescript
// ObjectId to string const userId: string = user._id.toString() // Populate with type const post = await Post.findById(id) .populate<{ author: IUserDocument }>('author') // post.author.name ab direct accessible hai! // Service layer pattern (controller se mongoose alag rakho) export const getUserById = async ( id: string ): Promise<IUserDocument | null> => { return User.findById(id).select('-password') }

Common Errors & Fixes

Ye errors sab beginners ko milte hain. Sab ke fixes yahan hain.

Top 5 Errors

1. Parameter 'x' implicitly has an 'any' type

❌ Problem
function greet(name) { return `Hello ${name}` } // ERROR: name ka type nahi bataya
✅ Fix
function greet(name: string): string { return `Hello ${name}` } // Type explicitly likho

2. Object is possibly 'null' or 'undefined'

❌ Problem
const user = users.find(...) console.log(user.name) // ERROR: user undefined ho sakta hai
✅ Fix
if (user) { console.log(user.name) } // ya optional chaining: console.log(user?.name)

3. Cannot find declaration file for module

❌ Problem
import express from 'express' // ERROR: Could not find // declaration file for 'express'
✅ Fix
npm install -D @types/express npm install -D @types/cors npm install -D @types/node // @types/ packages install karo

4. Mongoose _id is not string

❌ Problem
const user = await User.findById(id) const userId: string = user._id // ERROR: _id ObjectId type hai
✅ Fix
const userId: string = user._id.toString() // .toString() call karo // ObjectId → string
